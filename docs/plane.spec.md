Plane.so의 내부 구성은 크게 사용자가 체감하는 **제품 계층 구조(Product Hierarchy)**와 시스템을 지탱하는 **기술적 아키텍처(Technical Stack)** 두 가지 관점으로 나눌 수 있습니다.

오픈소스 기반의 프로젝트 관리 툴인 만큼 구조가 매우 체계적입니다.

---

### 1. 제품 계층 및 주요 컴포넌트 (Logical Structure)
Plane은 데이터를 관리하기 위해 다음과 같은 위계질서를 가지고 있습니다.

| 컴포넌트 | 설명 | 비고 |
| :--- | :--- | :--- |
| **Workspace** | 최상위 컨테이너. 조직이나 회사 단위의 모든 데이터를 담는 공간입니다. | URL의 식별자가 됨 |
| **Project** | 워크스페이스 내부의 개별 프로젝트 단위입니다. | 팀별/목표별 분리 |
| **Issue** | 가장 기본이 되는 작업 단위입니다. (상태, 우선순위, 라벨 포함) | 핵심 데이터 포인트 |
| **Cycle** | 일정 기간(Sprint) 동안 완료해야 할 이슈들의 묶음입니다. | 애자일 방법론 대응 |
| **Module** | 프로젝트 내의 특정 기능이나 영역별로 이슈를 그룹화합니다. | 서브 프로젝트 대안 |
| **Pages** | 프로젝트 관련 문서나 가이드를 작성하는 Wiki 기능입니다. | Notion과 유사 |
| **Views** | 필터링 및 정렬 조건을 저장해 둔 사용자 정의 화면입니다. | 칸반, 리스트, 캘린더 등 |

---

### 2. 기술적 아키텍처 및 시스템 구성 (Technical Architecture)
Plane은 현대적인 풀스택 프레임워크를 기반으로 구축되어 있으며, 주요 구성 요소는 다음과 같습니다.

* **Frontend (Next.js):**
    * 사용자 인터페이스를 담당하며, 반응형 웹 디자인을 제공합니다.
    * 상태 관리를 위해 `SWR` 또는 내부 API 핸들러를 사용합니다.
* **Backend (Django):**
    * Python 기반의 Django REST Framework(DRF)를 사용하여 강력한 API 서버를 구축했습니다.
    * 비즈니스 로직, 권한 관리, 데이터 검증을 수행합니다.
* **Database (PostgreSQL):**
    * 모든 프로젝트 데이터와 이슈 기록을 저장하는 메인 관계형 데이터베이스입니다.
* **Cache & Real-time (Redis):**
    * 데이터 캐싱 및 실시간 알림 처리를 담당합니다.
* **Storage (S3 / MinIO):**
    * 업로드된 이미지, 파일 등 미디어 데이터를 저장합니다.
* **Infrastructure (Docker):**
    * 전체 시스템은 컨테이너화되어 있어 클라우드(Self-hosted) 환경에서 쉽게 배포 및 확장이 가능합니다.

---

### 3. 특화된 내부 기능
* **Analytics:** 프로젝트의 진척도(Burn-down chart 등)를 시각화하는 분석 엔진입니다.
* **Drive:** 프로젝트별 파일을 관리하는 중앙 집중식 저장소 컴포넌트입니다.
* **Integrations:** GitHub, Slack 등 외부 툴과 데이터를 동기화하는 연결 모듈입니다.

Plane을 직접 설치(Self-hosting)하여 사용하실 계획인가요, 아니면 클라우드 버전의 기능적인 구성이 궁금하신 건가요? 상황에 맞춰 기술적인 설정법이나 상세 활용법을 더 설명해 드릴 수 있습니다.
